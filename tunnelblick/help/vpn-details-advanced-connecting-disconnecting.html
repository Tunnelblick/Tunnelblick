Tunnelblick Advanced Settings Window


Tunnelblick's Advanced Settings Window contains less-used settings for the configuration that is selected in the VPN Details Window.Â 


Most users do not need to change the settings in this window.


This window has two tabs. One tab contains advanced controls for when Tunnelblick is connecting or disconnecting the configuration; the other tab contains controls that are used while a connection is active.


Connecting and Disconnecting Tab


Each setting in this tab is controlled by a checkbox:


  • Scan configuration file for problems before connecting. When checked, Tunnelblick will scan the OpenVPN configuration file for problems before each attempt at connecting.
  • Use Tunnelblick tun/tap drivers: Tunnelblick contains its own tun/tap drivers. When checked, Tunnelblick will load the appropriate driver before attempting to make a connection if the driver is not already loaded. The driver will be unloaded until all configurations using the driver have been disconnected.
  • Flush DNS cache after connecting or disconnecting. When checked, Tunnelblick will flush the system's DNS cache after connecting and after disconnecting. This allows the system to use the appropriate DNS settings while the VPN connection is active and after it is disconnected.
  • Prepend domain name to search domains: This checkbox is only available when the default DNS/WINS setting of 'Set nameserver' is selected. When checked, Tunnelblick will add the domain name to the start of the search domain list if the list has not been specified manually. If the search domain list has been specified manually (in the Network System Preference Advanced window DNS tab), then nothing is done. When not checked on OS X 10.4 and 10.5, the domain name is added to the end of the search domains list. When not checked on OS X 10.6 and higher, the domain name replaces the search domains list unless search domains were entered manually, in which case nothing is done.


Two checkboxes control what Tunnelblick does when a system "Fast User Switch" occurs:


  • Disconnect when user switches out. When checked, Tunnelblick will disconnect the configuration when the user is switched out.
  • Reconnect when user switches in. When checked, Tunnelblick will reconnect the configuration when the user is switched back in, if the configuration was connected before the user switched out.


While Connected Tab


This tab contains a checkbox to control whether or not the network settings are monitored for changes while a connection is active.

It also contains two sets of actions that will be taken when particular network settings change in the following ways:

  • When change to pre-VPN value. These actions are taken when a network setting changes to the value it was before the VPN was established.
  • When changes to anything else. These actions are taken when a network setting changes to any other value.

One of three actions may be selected for each change:

  • Ignore. The network setting change will be ignored.
  • Restore. The network setting will be restored to the value it had after establishing the VPN.
  • Restart connection. The network setting change will cause the connection to be restarted.
